# UI Design

## Tujuan
Membuat rancangan halaman, navigasi, form, dan wireframe sederhana berdasarkan requirement.

## Kapan Digunakan
Digunakan setelah requirement dan bagian utama aplikasi diketahui.

## Input
- `03-requirement-specification.md`
- `06-architecture-design.md`
- `07-api-design.md`
- `profil-pengguna.md`

## Langkah Kerja
1. Baca input.
2. Identifikasi jenis pengguna dan tugas utama.
3. Buat daftar halaman aplikasi.
4. Tentukan navigasi antar halaman.
5. Rancang form, tabel, tombol, dan pesan validasi.
6. Buat wireframe teks sederhana untuk halaman penting.
7. Periksa hasil.
8. Berhenti jika informasi tidak cukup.

## Output
- `08-ui-design.md`
- `08-wireframe.md`

## Aturan
- Jangan membuat fakta baru.
- Tandai asumsi dengan label `ASUMSI`.
- Gunakan ID requirement yang sudah ada.
- Jangan membuat halaman di luar scope.
- Setiap form harus memiliki tujuan yang jelas.

## Quality Check
- Setiap halaman terhubung ke kebutuhan pengguna.
- Navigasi mudah diikuti.
- Form memiliki field dan validasi.
- Wireframe menunjukkan struktur halaman.
- UI tidak meminta data yang tidak diperlukan.

## Kondisi Gagal
- Jenis pengguna tidak jelas.
- Requirement tidak menjelaskan alur pengguna.
- Data yang harus ditampilkan atau diisi belum diketahui.

## Human Review
Manusia harus memeriksa apakah rancangan UI mudah digunakan oleh target pengguna.
